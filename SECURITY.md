# Security Policy

## Supported Versions

Security updates are provided for the latest stable version of **Market Forge**.

| Version        | Supported |
| -------------- | --------- |
| Latest release | ✅ Yes     |
| Older versions | ❌ No      |

## Reporting a Vulnerability

The security of **Market Forge** is important. If you discover a potential security vulnerability, please **do not create a public GitHub issue**, as this may expose details before a fix is available.

Instead, report the issue privately by sending an email to:

**[abhisachdeva0711@gmail.com](mailto:abhisachdeva0711@gmail.com)**

Please include the following information:

* A clear description of the vulnerability
* Steps to reproduce the issue
* Affected component or feature
* Potential impact
* Screenshots, logs, or proof of concept (if applicable)
* Suggested fix or mitigation (optional)

## Response Process

After receiving a vulnerability report:

1. The report will be reviewed and acknowledged as soon as possible.
2. The issue will be investigated and validated.
3. A fix will be developed and tested.
4. Security updates will be released when appropriate.

## Responsible Disclosure

Please allow reasonable time for investigation and remediation before publicly disclosing any vulnerability details.

Thank you for helping improve the security of **Market Forge** and protecting its users.
